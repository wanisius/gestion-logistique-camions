Bookmarklet — Extraction Commandes ADV

Sage X3 Web → PresenceCamions Android

① Installation du bookmarklet

⬇ Glissez ce bouton dans la barre des favoris du navigateur

Ou clic droit sur le bouton → Ajouter aux favoris

📶 Extraire ADV Sage X3

Alternative : copiez le contenu du fichier bookmarklet.txt (dans ce dossier) et collez-le comme URL d'un nouveau favori.

Le bookmarklet fonctionne sur Chrome, Firefox et Edge. Il ne nécessite aucune installation de logiciel.

② Utilisation pas à pas

  1. Ouvrez Sage X3 dans votre navigateur et naviguez vers la liste des Commandes de vente (écran GESSDH).
  2. Appliquez vos filtres habituels (période, statut, site…) pour afficher les commandes voulues. Assurez-vous que toutes les lignes sont visibles à l'écran.
  3. Cliquez sur le favori 📶 Extraire ADV Sage X3 dans votre barre de favoris.
  4. Une fenêtre apparaît avec un aperçu des données détectées et des listes déroulantes pour mapper les colonnes.
  5. Vérifiez le mapping, sélectionnez le type de transport par défaut, puis cliquez ⬇ Télécharger CSV.
  6. Transférez le fichier commandes_adv_YYYY-MM-DD.csv sur l'appareil Android (USB, email, Google Drive…).
  7. Dans PresenceCamions, connectez-vous en tant qu'ADV, puis utilisez le bouton Importer commandes.
💡 Cliquer à nouveau sur le bookmarklet (ou en dehors de la fenêtre) la ferme sans rien exporter.

③ Format du fichier CSV généré

Fichier UTF-8 avec BOM (compatible Excel), séparateur point-virgule :

client;destination;tonnage;dateLivraison;typeTransport
DUPONT SA;Casablanca;14T;2026-03-20;Location
ATLAS DIST;Marrakech;25T;2026-03-22;LUI-MEME
ColonneDescriptionTransformation automatique
clientRaison sociale / code tiersAucune
destinationAdresse ou ville de livraisonAucune
tonnageQuantité / tonnage commandéAjout du suffixe T si absent (14 → 14T)
dateLivraisonDate de livraison prévueConversion DD/MM/YYYY → YYYY-MM-DD
typeTransportMode de transportLOCATION → Location ; valeur par défaut si non mappé

⑤ Dépannage

ProblèmeSolution
« Aucune donnée trouvée »Assurez-vous d'être sur la liste GESSDH (pas sur une fiche individuelle). Faites défiler la liste pour charger les lignes si Sage X3 utilise un chargement lazy.
Colonnes mal détectéesAjustez manuellement le mapping dans la fenêtre du bookmarklet avant d'exporter.
Dates incorrectesVérifiez le format affiché dans Sage X3 (DD/MM/YYYY est le défaut FR/MA). Le bookmarklet convertit automatiquement vers YYYY-MM-DD.
Caractères spéciaux corrompusLe fichier est en UTF-8 avec BOM. Si Excel l'ouvre mal : Données → Depuis texte/CSV → choisir UTF-8.
Bookmarklet bloqué (CSP)Certains serveurs Sage X3 bloquent les bookmarklets. Solution : ouvrez la console navigateur (F12) et collez directement le contenu de source.js.
Mauvaises lignes sélectionnéesSi le bookmarklet capte des lignes de menu ou d'en-tête, mappez uniquement les colonnes utiles — les lignes sans client ni destination sont automatiquement ignorées.
